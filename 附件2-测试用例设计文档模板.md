# 测试用例设计文档

## 一、单元测试用例

主要针对 Service 层中分配到的 `UserServiceImpl`, `OrderServiceImpl`, `OrderVoServiceImpl` 进行单元测试。使用了等价类划分法、边界值分析法进行测试用例设计，以覆盖各个业务分支分支。基于 JUnit 5 与 Mockito 框架完成代码。

### 1. UserServiceImpl 测试用例

- **测试对象**：`com.demo.service.impl.UserServiceImpl`
- **测试脚本**：`src/test/java/com/demo/service/impl/UserServiceImplTest.java`

| 用例编号 | 测试函数 | 用例描述 | 预期结果 | 测试结果 | 结论 |
|----------|----------|----------|----------|----------|------|
| U-01 | `findByUserID` | 查询存在的 UserID | 返回对应的 User 对象 | 返回预期 User 对象 | 正确 |
| U-02 | `findById` | 查询存在的 ID | 返回对应的 User 对象 | 返回预期 User 对象 | 正确 |
| U-03 | `findByUserID`(Pageable) | 分页查询非管理员用户 | 返回含有用户列表的 Page | 返回预期 Page | 正确 |
| U-04 | `checkLogin` | 用户名和密码完全匹配成功 | 返回该用户 User 对象 | 返回该用户 User 对象 | 正确 |
| U-05 | `checkLogin` | 用户名存在但密码错误 | 返回 Null | 返回 Null | 正确 |
| U-06 | `create` | 传入有效 User 对象进行创建 | 返回模拟保存后的用户总记录数 | 返回预期总数 | 正确 |
| U-07 | `delByID` | 删除指定存在/不存在的 ID | 调用 dao 的 delete 方法 | 成功调用 | 正确 |
| U-08 | `updateUser` | 传入修改后的 User 实体 | 调用 dao 的 save 方法 | 成功调用 | 正确 |
| U-09 | `countUserID` | 查询对应 UserID 目前的数量 | 返回记录数量 | 返回预期数量 | 正确 |

### 2. OrderServiceImpl 测试用例

- **测试对象**：`com.demo.service.impl.OrderServiceImpl`
- **测试脚本**：`src/test/java/com/demo/service/impl/OrderServiceImplTest.java`

|----------|----------|----------|----------|----------|------|
| O-01 | `findById` | 查询已存在订单ID | 返回预期 Order 对象 | 返回预期 Order 对象 | 正确 |
| O-02 | `findDateOrder` | 在给定场馆和时间区间查询 | 返回查找到的 Order 列表 | 返回预期 Order 列表 | 正确 |

| O-04 | `updateOrder` | 修改场馆、时间、用户等参数 | 订单状态更新为未审核，并重算实付保存 | 成功保存并更新属性 | 正确 |
| O-05 | `submit` | 提交包含有效场馆的新订单 | 验证生成的新订单被构造和保存 | 成功保存新订单 | 正确 |
| O-06 | `delOrder` | 删除某确定的订单ID | 调用 dao 的 delete 方法 | 成功清理 | 正确 |

| O-08 | `confirmOrder` | 确认一个无效或空订单ID | 抛出 RuntimeException(订单不存在)| 抛出预期异常 | 正确 |
| O-09 | `finishOrder` | 完结已审核的 WAIT 状态订单 | 订单状态更新为 FINISH 并更新 | 状态改变并保存成功 | 正确 |
| O-10 | `rejectOrder` | 拒绝已存在的待审核订单 | 订单状态更新为 REJECT 并更新 | 状态改变并保存成功 | 正确 |
| O-11 | `findNoAuditOrder` | 获取当前未经过审核的订单分页 | 返回未审核订单分页 | 获取成功 | 正确 |
| O-12 | `findAuditOrder` | 查询处于 FINISH 或 WAIT 状态订单| 返回包含已审核订单的列表 | 获取成功 | 正确 |
